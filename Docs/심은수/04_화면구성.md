# 화면 구성

피그마 목업을 기준으로 각 화면을 정리, 실제 앱의 플로우와 기능 단위 패키지 구조를 대응

---

## 상위 구조

```
com.ssafy.a602
├── common
│   ├── di
│   ├── navigation
│   ├── network
│   └── ui
├── auth
├── home
├── learning
├── game
└── navbar
```

---

## 폴더별 설명

### 1. common/

여러 기능에서 공통으로 사용하는 코드들이 포함된다.

* di : 의존성 주입 모듈 (Hilt, Dispatcher, Logger 등)
* navigation : NavHost, Route, BottomNavItem 정의
* network : Retrofit/OkHttp 설정, Interceptor, 에러 처리
* ui : 공통 UI 컴포넌트, Theme, Color, Shape 등 디자인 시스템

공통 자원을 한 곳에서 관리해 기능 단위 코드와 분리한다.

### 2. auth/

로그인, 회원가입 등 인증 관련 기능을 담당한다.

* 로그인 화면: 이메일, 비밀번호 입력 후 로그인 처리
* 회원가입 화면: 닉네임, 아이디, 비밀번호 입력 및 검증

### 3. home/

앱 접속 시 처음 보이는 홈 화면을 관리한다.

* 오늘의 학습 추천, 최근 학습 이어하기, 게임/챗봇 진입 버튼
* 사용자 맞춤형 인사 문구 표시

### 4. learning/

수어 학습 전반을 담당하는 핵심 기능 패키지이다.

* 학습 메인: 학습 진행 현황, 오늘의 학습 표시
* 기초 학습: 자모 학습과 반복 퀴즈
* 테스트: 퀴즈 풀이 후 정오답 피드백 및 결과 표시
* 로드맵: Day 단위 학습 경로 제공 (일일 학습, 노래 학습)
* Day 학습: 단어 및 문장 단위 학습, 연습 퀴즈 진행

### 5. game/

리듬게임을 통한 학습 기능을 제공한다.

* 노래 목록: 선택 가능한 노래 리스트 표시
* 게임 플레이: 카메라 입력 기반 실시간 판정 (Perfect, Miss)
* 게임 결과: 점수, 판정 통계, 정확도 표시
* 게임 순위: 전체 사용자 간 랭킹 표시

### 6. navbar/

하단 네비게이션 바를 구성한다.
피그마 기준으로 탭은 다음과 같다:

* 검색
* 학습
* 챗봇
* 게임
* 마이페이지

---

## 주요 화면 흐름

1. 로그인/회원가입(auth) → 홈(home)
2. 홈에서 버튼을 통해 학습, 게임, 챗봇, 검색, 마이페이지 진입
3. 학습 진행 시 기초 학습 → 테스트 → 결과 → 로드맵(다음 학습)으로 이어짐
4. 게임은 노래 선택 → 카메라 판정 → 결과 및 순위 확인으로 이어짐

---

## 정리

* auth: 사용자 인증 기능
* home: 대시보드, 진입점 제공
* learning: 기초 학습, 테스트, 로드맵, 퀴즈
* game: 리듬게임, 결과 및 순위
* navbar: 하단 네비게이션 (검색, 학습, 챗봇, 게임, 마이페이지)
* common: 전역적으로 쓰이는 인프라 및 디자인 시스템

앱은 Jetpack Compose 기반 단일 Activity 구조이며, 각 기능은 NavGraph를 통해 화면 전환이 이루어진다.
