# AI 모델별 장단점 및 효용 정리
1. 모션 추출: MediaPipe

Google이 개발한 멀티모달 비전 라이브러리로, 포즈·손·얼굴 랜드마크 추출에 특화됨.

장점

온디바이스 실시간 처리: CPU 환경에서도 빠른 추론 가능

손·포즈·얼굴 지원: 수어에서 필요한 다중 랜드마크를 제공

가벼움: 설치 및 활용이 간단하며, 모바일·웹에도 적용 용이

오픈소스: 무료 사용 가능

단점

추상화된 좌표만 제공: 실제 수어 단어/문장 인식은 별도 모델 필요

제스처 복잡성 한계: 손가락 세부 움직임·표정 등의 정밀도는 떨어질 수 있음

환경 민감: 조명, 카메라 화질, 배경 등에 따라 성능 편차 발생

2. 수어 인식: YOLO (You Only Look Once)

객체 탐지(Object Detection)에 특화된 딥러닝 모델로, 수어 인식을 위한 손 모양/제스처 분류에 활용.

장점

실시간 탐지 가능: FPS가 높아 빠른 응답 필요 서비스에 적합

범용성: 손 모양, 제스처 등을 객체로 간주하여 인식 가능

Custom Dataset 학습 용이: 수어 데이터셋을 레이블링 후 학습 가능

다양한 버전 선택지: YOLOv5, v7, v8, Nano 버전 등 상황별 최적화

단점

포즈 이해 부족: 손 모양 단일 탐지는 강점이나, 문맥·문장 단위 이해는 어려움

데이터 의존적: 정확한 인식을 위해 대규모 수어 데이터셋 필요

표정·입술 패턴 반영 한계: 수어의 의미적 요소(표정, 구강 움직임 등)는 별도 모듈 필요

3. 대화: LLM (Large Language Model)

수어 → 텍스트로 변환된 결과를 기반으로 대화/응답 생성을 담당.

장점

자연스러운 문장 생성: 수어에서 변환된 단어/문장을 문법적으로 매끄럽게 보정

컨텍스트 이해: 사용자의 대화 맥락을 유지하며 답변 가능

확장성: 도메인 특화 지식(RAG, Fine-tuning)으로 커스터마이징 가능

단점

모델 크기 문제: 대규모 모델은 서버 비용 및 지연(latency) 증가

Hallucination 가능성: 잘못된 답변 생성 가능 → RAG/검증 모듈 필요

학습 비용 부담: 도메인 특화 파인튜닝 시 리소스 소모 큼

4. 대화용 LLM 학습/추천
(1) 학습 방식

Fine-tuning: 특정 도메인(수어 관련 대화)에 맞춰 추가 학습

Instruction Tuning: 수어 사용자 맞춤형 응답 스타일 학습

RAG (Retrieval-Augmented Generation): 외부 지식 DB 연동, 최신 정보 반영

(2) 추천 모델

오픈소스 모델

LLaMA 3 (Meta): 고성능, 한국어 지원 양호, 파인튜닝/LoRA 용이

Mistral 7B/8x7B: 경량 + 성능 균형, 대화 응용에 적합

Qwen 2 (Alibaba): 한국어·중국어·영어 다국어 지원 강점

Upstage Solar (embedding + LLM): 한국어 최적화, RAG와 연동 적합

상용 API 기반

OpenAI GPT-4o/5: 안정적 성능, 멀티모달 확장성

Anthropic Claude 3.5: 긴 맥락 처리 강점, 안전성 우수

Google Gemini 1.5/2.0: 멀티모달 대화, 비전+텍스트 결합 가능

5. 전체 파이프라인 예시
flowchart LR
    A[사용자 수어 입력] --> B[MediaPipe: 포즈·손 랜드마크 추출]
    B --> C[YOLO: 손 모양/제스처 탐지 → 수어 단어]
