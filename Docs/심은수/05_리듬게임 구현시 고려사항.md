# 리듬게임 오디오·동기화·카메라/ML 설계 요약서

피그마 목업을 기준으로, 리듬게임 구현에 필요한 핵심 기술을 “온디바이스”와 “서버 AI” 두 운용 모드로 구분해 정리했다. 범위는 오디오 파이프라인, 타이밍·동기화, 카메라/ML 파이프라인의 큰 틀이다.

---

## 1. 오디오 파이프라인(Media3/ExoPlayer)

### 1.1 공통 목표와 구성

* 오디오 타임라인을 단일 기준으로 사용한다.
* 구성 요소: ExoPlayer 래퍼(재생/일시정지/시크/위치 스트림), 버퍼 게이트(준비+여유 버퍼 확보 후 카운트다운 시작), 오류 정책(재시도·사용자 피드백), 드리프트 모니터(시간 편차 관찰).

### 1.2 온디바이스 모드

* 시간 기준: `player.currentPosition()`만 사용.
* 특성: 네트워크와 무관, 지연 변동이 작다.
* 운영 포인트

  * 위치 스트림은 16\~32ms 간격으로 발행해 UI·판정에 사용.
  * BUFFERING 진입 시 판정 루프는 일시 정지, READY 복귀 시 현재 위치로 차트 인덱스를 재정렬.
  * 오디오 포커스 변화(이어폰 분리 등) 시 재생 상태와 판정 루프를 동기화.

### 1.3 서버 AI 모드

* 시간 기준: 오디오는 로컬, 인식 이벤트는 서버가 부여한 추론 시각을 사용.
* 시차 요소: 업로드/대기/추론/응답의 합산 지연.
* 운영 포인트

  * 서버 응답에 클라이언트 송신시각·서버 수신/추론시각을 포함해 왕복지연과 시계차를 추정한다.
  * 시작 시 플레이를 지연(예: p95 지연만큼)하거나, 판정 배지 표시에만 보정값을 적용하는 전략 중 하나를 선택한다.
  * 지연이 큰 구간은 유예 상태로 처리하고, 유예 초과 시 Miss 확정.

## 2. 타이밍·동기화 전략

### 2.1 공통 원칙

* 최종 판정은 오디오 타임라인 기준으로 통일한다.
* 드리프트(누적 오차)와 지터(변동성)를 분리해 대응한다.
* 카운트다운 동안 인식 이벤트는 폐기해 워밍업 잡음을 제거한다.

### 2.2 온디바이스 모드

* 인식 이벤트 타임스탬프는 프레임 캡처 시각이 아니라 “방출 직전의 오디오 위치”로 표기한다.
* 5\~10초 간격으로 평균 편차를 관찰해 임계치 초과 시 상수 오프셋으로 보정한다.
* BUFFERING 구간에서 들어온 인식 이벤트는 폐기하고, 재개 시 차트 인덱스를 현재 위치로 재탐색한다.

### 2.3 서버 AI 모드

* 시계 동기화: 간단한 NTP 또는 왕복지연 기반 약식 동기화로 서버–단말 시간차를 추정한다.
* 투영: 서버 추론시각을 로컬 기준으로 변환해 오디오 타임라인에 정렬한다.
* 지터 완화: 최근 이벤트의 중앙값/지수평활로 순간 변동을 완화하고, 보정된 시간에 맞춰 UI 피드백을 스케줄링한다.
* 대량 지연·누락 시 일시정지→재동기화→재개 경로를 안내한다.

## 3. 카메라/ML 파이프라인

### 3.1 공통 구성

* CameraX: Preview + ImageAnalysis, 최신 프레임 유지 전략 사용.
* 목표 해상도·FPS: 720p 기준 15\~24fps.
* 전처리: YUV→RGB, 필요 시 ROI 크롭, 리사이즈·정규화. 후처리: 임계치, 중복 방지, 스무딩.

### 3.2 온디바이스 모드

* 추론은 TFLite/MediaPipe(필요 시 GPU/NNAPI)로 수행한다.
* 성능 운영: Analyzer 전용 스레드, 버퍼 재사용으로 GC 최소화, p95 추론 지연 80\~120ms 목표.
* 타임스탬프는 이벤트 방출 직전 오디오 위치로 기록해 오디오 기준과 직접 정렬한다.
* 실패·폴백: Delegate 단계적 다운그레이드 후 불가 시 서버 모드 전환을 안내한다.

### 3.3 서버 AI 모드

* 업로드 단위 선택

  * 원시 프레임 업로드: 서버 유연성은 높지만 대역폭·지연 비용이 큼.
  * 키포인트 업로드(권장): 단말에서 경량 추출(예: MediaPipe) 후 좌표만 전송해 대역폭·개인정보 부담을 줄인다.
* 전송/신뢰성: QUIC/UDP(지연 최소) 또는 HTTP/2(신뢰성). 업링크 상태에 따라 업로드 FPS를 단계적으로 낮춘다.
* 보안/개인정보: TLS, 토큰 인증. 원시 영상 전송 시 명시적 동의와 자동 폐기 정책을 고지한다.
* 타임스탬프: 패킷에 단말 송신시각을 포함하고, 서버가 추론시각을 부여한 뒤 클라이언트가 로컬 시간으로 투영한다.

---

## 결론

어떤 모드이든 오디오 타임라인을 기준으로 일관된 타임스탬프 처리와 버퍼링 전이 대응을 갖추면 Perfect/Miss 중심의 판정 UX를 안정적으로 제공할 수 있다.
