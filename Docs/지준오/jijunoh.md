## 미디어 데이터 통신(스트리밍) 방법 설계 클라이언트 ↔ 백엔드

### 방식 1 : WebSocket을 통한 바이너리 스트리밍 데모 구현

- 프론트
    - 웹캠 녹화(MediaRecorder) → Chunk 단위로 자름 → 인코딩 → 파일 전송
- 백엔드
    - 파일 수신 → 디코딩 → Buffer에 쌓음 → Buffer를 Stream으로 연결 → 모델에 입력 → 결과값 출력

⇒ 단점

1. 프론트와 백에서 인코딩 및 디코딩 과정에서 부하가 너무 큼
2. 지연성이 떨어짐 Chunk 만큼의 지연 + @
3. 복잡도가 높음

### 방식 2 : RTMP / HLS / MPEG-DASH

- 아직 시도 안해봄
- 우려되는점
    - 해본적이 없음

### 방식 3 : WebRTC

- 아직 시도 안해봄
- 우려되는점
    - 복잡도 높음
    - 연결 이후 계속해서 네트워크 부하 생김
    - 파이썬 백엔드에서 WebRTC를 구현해야 함
